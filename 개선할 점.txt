문제 -> 해결

1. 노트를 거의 생성하지 않음 
	-> 노트가 있는데 없다고 예측한 경우 로스를 크게 주어 개선됨

2. 노트가 시작되는 지점의 근처에 산발적으로 노트가 있다고 예측 
	-> 여러 방법으로 해당 지점의 주변까지 로스를 크게 주었지만 전혀 개선되지 않음
	-> 산발적으로 나타나는 노트를 하나만 남기도록 후처리 (예정)
	-> 순방향으로의 주파수 변동에 집중하도록 양방향RNN을 단방향RNN으로 전환하니 성능이 매우 안좋아짐
	-> 입력에 따라 달라지는 가중치로 여러 종류의 드럼을 파악하고 어텐션으로 맥락을 볼 수 있는 트랜스포머 모델로 변경 (예정)

3. 과적합 -> validation set을 만들어서 early stopping

4. 모델이 노트의 종류를 헷갈려함
	-> 스펙트로그램의 freq. bin을 더 촘촘하게 변경하여 하이햇, 라이드 구분 성능이 확실히 좋아짐
	-> 학습스케줄링: 로스함수를 변경하며 학습 (예정*)
	-> 짧은 어택에 집중하게 하기위해 rnn의 hidden_size를 감소시켰으나 언더피팅 발생
	-> depth-wise conv 사용하여~
	
5. 하나의 타임스텝에 많은 노트가 있다고 예측함
	-> 로스 가중치를 노트가 존재하는 타임스텝 전체에 적용하여 동시에 여러 노트를 예측하는 걸 해소하였지만 
		모델이 노트가 존재하지 않는 쪽으로 예측하도록 학습됨. 그리고 박자도 좀 이상해짐.	
		fine-tuning 으로도 시도해봤지만 learning rate가 높아서 그런지 같은 모델로 수렴하는 듯 하였음.

	-> 미디 생성 시, 중복되면 안되는 노트들을 고려하면서 생성 ex) 킥-스, 스-탐, 하-라 (예정*)

6. 1에폭 차이인데 출력에서 차이가 큼
	-> 배치사이즈 증가 (예정)

7. 일반화성능 높히기
	-> 학습데이터에 랜덤하게 노이즈랑 음량변화를 부여
